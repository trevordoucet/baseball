Project Description:

Our project aims to look at MLB pitching data in order to find relationships between the various features that coexist between a pitcher's technique and strategy, as well as the attributes that best predict a strike in the major leagues. We will be focusing on variables related to, but not limiteed to: a pitcher's release position, spin rate, zone location, and the hitter's outcome.

Questions that we attend to address with Machine Learning and Statistical Analysis:

  • What are the most prominent features that predict a strike?
  • Can we identify a pitcher that excels with these features?
  • What pitchers best resemble this identified pitcher?

Dataset

Our primary dataset will come from https://baseballsavant.mlb.com/. This website provides official and comprehensive data from all MLB games. The main challenges we faced involved the manual process for downloading the raw csv files.

The secondary dataset will come from https://databases.usatoday.com/major-league-baseball-salaries-2023/. The plan will be to webscrape the tables containing contract information as baseball salaries are difficult to obtain casually.

Some useful context about pitching stats:

There are a number of factors that go into what makes one pitch different from another. Factors such as release speed, spin rate, and the type of pitch can affect the difficulty a batter will face when attempting to hit a thrown pitch. A factor that we will examine throughout this project is the "zone" of the pitch. This refers to the sectors within and around a given "strike-box" that umpires will use to determine if a pitch is a strike or a ball. There are 14 zones within this strike-box, with zones 1-9 in the center of the zone, and 10-14 along the perimeter.

Using a Mac

Our implementaion works with Safari as the main webdriver. If you want to run the below code and you have a mac: go to settings > advanced > show features for web developers (check box at the bottom) > developer > select "Allow remote automation" under automation.

How it works

The driver will load the desired website into a new Safari window. The code will then extract each table's information, append the details to a list, then find and click the "next" buttom. These operations work in a while loop and will continue until the last page has been reached. The whole process takes approximately 5 minutes

In[]:
# Set up Safari WebDriver
driver = webdriver.Safari()

# USA Today: Major League Baseball Salaries 2023
url = "https://databases.usatoday.com/major-league-baseball-salaries-2023/"
driver.get(url)

players = []

# Loop to go through all pages
while True:
    try:
        # Wait for the data table to load
        table = WebDriverWait(driver, 10).until(
            EC.presence_of_element_located((By.CLASS_NAME, "csp-data"))
        )

        # Extract rows from the current page
        rows = table.find_elements(By.TAG_NAME, "tr")
        for row in rows:
            cols = row.find_elements(By.TAG_NAME, "td")
            player_data = [col.text.strip() for col in cols]
            players.append(player_data)

        # Find and click the "Next" button
        next_button = WebDriverWait(driver, 10).until(
            EC.element_to_be_clickable((By.LINK_TEXT, "Next"))
        )
        next_button.click()
        
        # Wait for a short while to ensure the page loads
        time.sleep(3)

    except Exception as e:
        print("An error occurred or no more pages:", e)
        break

# Print the results
for player in players:
    print(player)

# Close the browser
driver.quit()

# DataFrame
scraped = pd.DataFrame(players)

# Export as a backup if you run this - use complete "salaries.xlsx"
scraped.to_excel('salaries_copy.xlsx')

Machine Learning Chosen: Random Forest

Why Random Forest

The Random Forest algorithm handles one-hot encoded categorical features effectively and does not require feature scaling, making it easy to integrate mixed data types. Random Forest supports balancing class weights which we found helpful for fine tuning the dataset's imbalance of strikes and balls. Resampling techniques were available to be applied to create equal samples in training, ensuring the model performs well for both classes.

Inputs:

The features we chose are a mix of categorical features and quanatative features. We chose what features we believed to be most important and decided to include a comprehensive list to reduce bias, knowing variability would be an obstacle.

The categorical features we chose to one-hot encode:

'pitch_number', 'outs_when_up', 'inning', 'strikes', 'balls', 'at_bat_number', 'zone'

The quantitative features used:

'release_speed', 'release_spin_rate', 'effective_speed', 'release_pos_y', 'release_extension', 'spin_axis'

Assumptions:

We made assumptions with the categorical features chosen. We believed that pitch type would have a stronger feature relevance, but scored amongst the most irrelevant in the results. Additionally, we assumed the one-hot encoded inning and zone, transformed appropriately to represent their contribution to the model. Lastly, we assumed the dataset does not contain signficant outliers.

Important inputs that we added to the RandomForestClassifier function was limiting the max depth and min samples split to 10 and 5, respectively. Shallower trees require fewer splits and nodes, reducing the computational cost for both training and inference. This prevented overfitting and reduced noise sensitivity which we found necessary given the large number of features chosen

Performance:

Overall the model's performance was satisfactory with an 81% accuracy, 95% precision on strikes, 95% recall for balls, and a macro average F1-score of 0.81.

# Encode the target variable

# Features
scenario_features = ['pitch_number', 'outs_when_up', 'inning', 'strikes', 'balls', 'at_bat_number', 'zone']
quantitative_features = ['release_speed', 'release_spin_rate', 'effective_speed', 'release_pos_y', 'release_extension', 'spin_axis']

# Transform for one-hot
for col in scenario_features:
    test_models_df[col] = test_models_df[col].astype('category')

# One-hot encode categorical features
categorical_encoded = pd.get_dummies(test_models_df[scenario_features], drop_first=True)

# Combine encoded categorical features with quantitative
features = pd.concat([categorical_encoded, test_models_df[quantitative_features]], axis=1)
target = test_models_df['type']

# Split the dataset

The code below generates a learning curve for the Random Forest model to evalute performance across different training test sizes. The learning curve function computes training and validation accuracy scores for the model using a 5-fold cross-validation across varying proportions of the training data. The mean and standard deviation of the training and validation scores are calculated to assess the variability in performance across folds.

# Learning curve data
train_sizes, train_scores, val_scores = learning_curve(
    estimator=randomf,
    X=features,
    y=target,
    cv=5, # 5-fold cross validation
    scoring='accuracy',
    n_jobs=-1,
    train_sizes=np.linspace(0.1, 1.0, 10) # 10 increments from 10% to 100% of the data
)

# Mean and standard deviation of scores
train_mean = np.mean(train_scores, axis=1)
train_std = np.std(train_scores, axis=1)
val_mean = np.mean(val_scores, axis=1)
val_std = np.std(val_scores, axis=1)

Conclusion

Based on our primary investigation, the most important factor of a pitch that results in a strike is the zone that it targets, moreso than other factors such as speed and spin rate. This indicates that pitchers that are successful in striking out batters are throwing towards a preferred zone.

From the results of our secondary investigation, no one pitcher shows dominance over multiple zones, indicating that a pitcher can show great value to a team by mastering a pitch targeting a certain zone within the strike-box. Drawing from this information, teams may find that there are pitcher out there that can perform similarly to the top pitchers in a given zone for a fraction of the cost. When watching baseball, this information can be useful in predicting where a given pitcher is going to attempt to throw based on their most dominant pitch type and zone.
